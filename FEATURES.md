# PingPong Bot — Описание реализованных бизнес-функций

## Обзор

PingPong Bot — Telegram-бот для организации и бронирования тренировок по настольному теннису. Система включает бот на базе Edge Function (webhook) и веб-дашборд для просмотра расписания.

---

## 1. Управление пользователями

### Регистрация
- Автоматическая регистрация при первом `/start` — создаётся запись в `bot_users` с данными из Telegram (имя, username, telegram_id).
- При повторном обращении данные обновляются (если изменились имя/username).

### Суперадмины
- Определяются через переменную окружения `SUPER_ADMIN_IDS` (список Telegram ID через запятую).
- Имеют доступ к созданию групп и полному администрированию.

### Профиль пользователя (`👤 Мой профиль`)
- Отображает: имя, username, количество групп, предстоящих тренировок, активных страйков.

---

## 2. Группы

### Создание группы (`/newgroup Название`)
- Доступно только суперадминам.
- Создаёт группу с дефолтными параметрами:
  - `max_participants`: 8
  - `freeze_hours`: 4
  - `timezone`: Europe/Moscow
- Автоматически генерирует `invite_code`.
- Создатель становится администратором и участником группы.

### Вступление в группу (deep link)
- По ссылке `t.me/бот?start=join_КОД` пользователь вступает в группу.
- Проверяется: не заблокирован ли пользователь, не является ли уже участником.

### Просмотр групп (`📋 Мои группы`)
- Список всех групп, в которых пользователь является участником (не забанен).

---

## 3. Расписание и сессии

### Шаблоны расписания (`/addschedule`)
- Формат: `/addschedule GROUP_ID ДЕНЬ НАЧАЛО КОНЕЦ`
- Дни: 0=Вс, 1=Пн, 2=Вт, 3=Ср, 4=Чт, 5=Пт, 6=Сб
- Пример: `/addschedule abc12345 1 19:00 21:00` — понедельник 19:00–21:00.
- Доступно администраторам группы и суперадминам.

### Автогенерация сессий
- На основе шаблонов расписания автоматически создаются конкретные тренировки на **14 дней вперёд**.
- Генерация происходит при просмотре расписания (on-demand) через `generateSessions()`.
- Используется `upsert` с конфликтом по `(group_id, date, start_time)` для идемпотентности.

### Просмотр расписания (`📝 Расписание`)
- Если пользователь в одной группе — сразу показывается расписание.
- Если в нескольких — сначала выбор группы.
- Для каждой сессии отображается:
  - Дата и время
  - Количество записанных / макс. мест
  - Количество в очереди
  - Статус пользователя (✅ записан / ⏳ в очереди)
  - Кнопки действий (записаться / отменить) с учётом заморозки.

---

## 4. Бронирование тренировок

### Запись на тренировку
- Нажатие кнопки `✅ Записаться` в расписании.
- Если есть свободные места → статус `active`.
- Если мест нет → статус `waitlist` с присвоением `waitlist_position`.

### Лист ожидания (FIFO)
- При отмене активной записи автоматически продвигается первый из очереди (`waitlist_position ASC`).
- Продвинутый пользователь получает уведомление в Telegram: «🎉 Место освободилось!».

### Отмена записи
- Пользователь видит кнопку `❌ Отменить` (только до заморозки).
- Подтверждение через промежуточный экран «Вы уверены?».
- При отмене бронь переходит в статус `cancelled`.

### Покинуть очередь
- Пользователь в листе ожидания может выйти из очереди (кнопка `❌ Покинуть очередь`).

### Заморозка (freeze)
- За `freeze_hours` часов до начала тренировки:
  - Обычные пользователи **не могут** записаться или отменить запись.
  - Автоматическое продвижение из waitlist **прекращается**.
- Значение `freeze_hours` настраивается для каждой группы (по умолчанию 4 часа).

---

## 5. Система страйков и банов

### Страйки
- Хранятся в таблице `strikes` с привязкой к пользователю, группе и (опционально) сессии.
- Имеют срок действия (`expires_at`).
- Администратор может просмотреть активные страйки группы.

### Баны
- Поле `is_banned` в `group_members`.
- Заблокированный пользователь не может:
  - Записаться на тренировку (видит сообщение с количеством страйков).
  - Вступить в группу повторно.

---

## 6. Администрирование

### Панель управления (`⚙️ Управление`)
- Доступна администраторам групп и суперадминам.
- Суперадмины видят кнопки: «Создать группу», «Все группы».

### Редактирование группы (`/editgroup`)
- Формат: `/editgroup GROUP_ID ПОЛЕ ЗНАЧЕНИЕ`
- Допустимые поля:
  - `name` — название группы (строка)
  - `max` — максимальное количество участников (число > 0)
  - `freeze` — часы заморозки перед тренировкой (число > 0)
  - `timezone` — часовой пояс (строка, например `Europe/Moscow`)
- Доступно администраторам группы и суперадминам.
- Пример: `/editgroup abc12345 max 10`

### Удаление группы (`/deletegroup`)
- Формат: `/deletegroup GROUP_ID`
- Доступно **только суперадминам**.
- При удалении каскадно удаляются:
  - Администраторы и участники группы
  - Шаблоны расписания
  - Все будущие бронирования отменяются (статус `cancelled`)
  - Сессии и страйки удаляются
- Пример: `/deletegroup abc12345`

### Управление группой (inline-кнопки)
- Отображает: количество участников, макс. мест, freeze_hours, инвайт-код.
- Разделы:
  - **📅 Расписание группы** — список тренировок с кнопками отмены.
  - **👥 Участники** — список с именами, username, статусом (✅/🚫).
  - **⚠️ Страйки** — активные страйки.
  - **🔗 Новая инвайт-ссылка** — генерация нового invite_code.
  - **✏️ Редактировать** — inline-меню для изменения параметров:
    - Макс. участников — выбор из предустановленных значений (4, 6, 8, 10, 12, 16, 20)
    - Заморозка — выбор из предустановленных значений (1, 2, 3, 4, 6, 8, 12, 24 часов)
    - Название / Часовой пояс — инструкция с готовой командой `/editgroup`
  - **🗑 Удалить** — удаление группы с подтверждением (только суперадмины).

### Отмена тренировки (админ)
- Подтверждение через промежуточный экран.
- Все записанные (active + waitlist) получают уведомление.
- Бронирования переводятся в `cancelled`.
- Сессия получает статус `cancelled`.

## 7. Веб-дашборд

### Главная страница
- **Заголовок** с иконкой 🏓 и названием «PingPong Bot».
- **Пустое состояние** (нет групп) — пошаговый гайд по настройке:
  1. Настройка Webhook
  2. Создание группы
  3. Добавление расписания
  4. Приглашение участников

### Список групп (левая панель)
- Карточки групп с первой буквой названия, макс. участниками и freeze_hours.
- Подсветка выбранной группы.

### Расписание (правая панель)
- Список предстоящих сессий с:
  - Датой (день недели + число) на русском языке.
  - Временем начала и окончания.
  - Прогресс-баром заполненности.
  - Бейджем очереди.
- Цветовая индикация: красный при полной заполненности.

## 8. Схема базы данных

| Таблица | Назначение |
|---|---|
| `bot_users` | Пользователи Telegram-бота |
| `groups` | Группы тренировок |
| `group_admins` | Администраторы групп |
| `group_members` | Участники групп (+ бан) |
| `schedules` | Шаблоны расписания (день недели + время) |
| `sessions` | Конкретные тренировки (дата + время + статус) |
| `bookings` | Бронирования (active / waitlist / cancelled) |
| `strikes` | Штрафы с истечением |
| `notifications_log` | Лог уведомлений |

### Ключевые enum-типы
- `booking_status`: `active` | `waitlist` | `cancelled`
- `session_status`: `scheduled` | `cancelled` | `completed`

## 9. Технический стек

- **Frontend**: React + Vite + TypeScript + Tailwind CSS + shadcn/ui
- **Backend**: Lovable Cloud (Edge Functions на Deno)
- **База данных**: PostgreSQL
- **Бот**: Telegram Bot API через webhook → Edge Function
- **Библиотеки**: date-fns, @supabase/supabase-js, recharts, react-router-dom
